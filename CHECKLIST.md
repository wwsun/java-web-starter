# Tech Marathon Check List

> [!tip] 核心原则
> Demo 能跑比代码优雅更重要。先打通主干，再补细节。

## 1. 功能完整性（P0 先打通主干）

不要并行开发，要先打通一条完整链路：

```
数据库建表 → 后端 CRUD + 核心约束 → API 联调 → 前端能跑通
```

主干通了之后再加筛选、分页、状态机等细节。

---

## 2. 设计文档与 Spec Driven（考前 30 分钟的投资）

拿到题目后，**先别急着写代码**，用 15–30 分钟：

1. 画出数据表结构（核心字段、关联关系）
2. 列出所有业务规则（特别是冲突/约束逻辑）
3. 写出 API 列表（`METHOD + Path + 请求/响应结构`）

> [!note]
> 可以用 Claude 帮你生成设计文档，放在 `doc/` 里。评分时这是高分项，且会倒逼你提前发现模糊点。

---

## 3. 技术实现质量（3 个细节最容易被扣分）

> [!warning] 高频失分点
>
> - **冲突检测必须加锁**：核心业务的并发写需要 `FOR UPDATE`（已有样例可参考 `BookingServiceImpl`）
> - **不要把 Entity 直接返回给前端**：用 `XxxVO.from(entity)` 转换
> - **Controller 不要 try-catch**：抛 `BusinessException`，统一由 `GlobalExceptionHandler` 处理

---

## 4. 可测试性（写一层就测一层）

- 后端每个 Service 方法写完就加单元测试 → 用 `@WebMvcTest` 覆盖 Controller
- 前端每个页面写完就补 `.test.tsx`（mock API，测三态：`loading` / `error` / `empty`）

评分时跑 `mvn test` 和 `npx vitest run`，测试全绿是加分信号。

---

## 5. AI 工具链使用（明牌加分项）

评分者会关注你是否高效使用 AI。实用打法：

| 任务                | 怎么用 Claude                                      |
| ------------------- | -------------------------------------------------- |
| 建表 SQL            | 描述业务规则，让 Claude 生成 + 审查索引是否合理    |
| 写 Service 业务逻辑 | 提供接口签名 + 规则，Claude 生成，你审查逻辑正确性 |
| 写测试              | 提供实现代码，让 Claude 补测试                     |
| 前端页面            | 描述交互，让 Claude 生成骨架，你填数据逻辑         |
| 卡壳排查            | 把报错直接贴进来，**不要自己 debug 超过 5 分钟**   |

> [!quote]
> 你的核心价值在于：**设计决策、规则正确性、联调验证**——而不是手写样板代码。

---

## 时间节奏参考（以 4 小时马拉松为例）

| 时间段      | 任务                                                       |
| ----------- | ---------------------------------------------------------- |
| 0–30 min    | 读题、设计数据库、写 API 列表 → 生成设计文档               |
| 30–90 min   | 后端：建表、写核心 Service + 冲突逻辑、写 Service 单测     |
| 90–150 min  | 后端 Controller + 前端主页面打通、用 `scripts/api.sh` 联调 |
| 150–210 min | 前端补全：筛选 / 分页 / 状态展示，补前端测试               |
| 210–240 min | 收尾：跑全量测试、清理 TODO，检查有没有裸奔的敏感信息      |

---

## 最容易丢分的坑

> [!danger] 提交前自查清单
>
> 1. 设计文档忘了写（评分时才意识到这是一个大项）
> 2. 测试写了但没跑，实际上有失败用例
> 3. 主干流程跑通了但边界规则没处理（比如：过去时间不能预约、取消后不能再操作）
> 4. `password` / `deleted` 字段出现在 API 响应中
> 5. 提交前没确认 `docker compose up -d mysql && mvn test` 能在干净环境跑通
